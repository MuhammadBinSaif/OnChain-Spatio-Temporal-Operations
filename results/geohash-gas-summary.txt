# Geohash Gas Benchmark

## Scenario

- Rounds: 10
- Contract: `Geohash.sol` only
- Parcels: 3
- Lots: 4
- Batches: 12
- Parcel declaration gaps: 10 days
- Batch creation gaps: 7 days
- Query time window: day offsets 56 to 91
- Area query target: the first parcel area (prefix4 of area A)

Gas for writes comes from transaction receipts. Gas for reads comes from `estimateGas`.

## Write Interpretation

- Exact average `registerBatch` gas in the main scenario: 680226.73
- Exact average measured setup gas inside `registerBatchMeasured`: 96569
- Exact average measured spatial-tree gas inside `registerBatchMeasured`: 115555.33
- Exact average measured temporal-tree gas inside `registerBatchMeasured`: 365244
- Exact average measured shared-tail gas inside `registerBatchMeasured`: 77625
- Base measured transaction gas when the leaf and time bucket already exist: 269722
- Base spatial-tree gas in that base case: 34720
- Base temporal-tree gas in that base case: 39402
- Additional temporal-tree gas when opening a new time bucket: 324417
- Additional spatial-tree gas when inserting a new leaf into an existing time bucket: 211808
- Measured transaction gas when both a new leaf and a new time bucket are introduced: 805947

The original `registerBatch` path is still used to report real total write gas.
The measured path `registerBatchMeasured` is used only to extract exact internal section costs for setup, spatial-tree writes, temporal-tree writes, and shared bookkeeping.
